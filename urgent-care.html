Urgent Care E/M Uplift Blueprint

MyPulseScan × Particle Health — Single-Encounter Data Fetch & Revenue Optimization

Model Focus: Immediate data exchange at check-in to support clinical safety and E/M data-review upcoding. This model targets per-encounter revenue — no long-term biometric tracking or device monitoring required.

Architecture & Data Flow

The transaction loop triggers automatically upon patient arrival, ensuring external data is parsed into the chart before triage begins.

[1. Front-Desk Check-In / Kiosk Arrival] │ ▼ [2. EHR Trigger: Patient Status → 'Arrived'] │ ▼ [3. Particle Health API Call (OAuth 2.0 / Mutual TLS)] │ ▼ Queries Carequality, CommonWell, Surescripts [4. Record Aggregation — C-CDA XML / FHIR JSON] │ ▼ [5. EHR Clinical Summary Panel Auto-Populated] │ ▼ [6. Provider Reviews External Data → Documents in Note] │ ▼ [7. E/M Code Justified: 99203→99204 (new) or 99213→99214 (established)]

E/M Uplift Economics

New Patient Uplift

99203 → 99204 = +$39.00/visit

Requires documented review of externally sourced records. Particle fetch + provider text macro = audit-proof justification.

Established Patient Uplift

99213 → 99214 = +$33.00/visit

Same documentation pattern. Higher volume — most urgent care visits are established patients returning for new complaints.
